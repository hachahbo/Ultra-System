# Frontend Master Rule

When working on frontend tasks, components, pages, or styling in this repository, you MUST utilize the following skills as appropriate for the task at hand to ensure high-quality, production-ready, and accessible UI/UX. To invoke a skill, actively read its instructions before proceeding.

- **Baseline UI Quality & Standards**:
  - `baseline-ui`: Validate animation durations, typography scales, component accessibility, and prevent layout anti-patterns.
  - `ui-ux-pro-max`: Leverage UI/UX design intelligence using predefined styles, color palettes, font pairings, charts, and layout stacks.

- **Component & Design System Engineering**:
  - `frontend-design`: Create distinctive, production-grade frontend interfaces with premium design quality, avoiding generic AI aesthetics.
  - `frontend-ui-engineering`: Build robust, production-quality UIs, implementing responsive layouts and managing frontend state effectively using Next.js/React Native best practices.
  - `shadcn`: Manage shadcn/ui components—adding, searching, fixing, debugging, styling, and composing UI.

- **Animations & Transitions**:
  - `transitions-dev`: Implement production-ready CSS transitions for interactive elements (e.g., dropdowns, modals, success checks, hover states).
  - `mastering-animate-presence`: Audit and implement Framer Motion `AnimatePresence` best practices for exit animations, modals, and presence states.

- **Layout & Responsiveness**:
  - `responsive-design`: Apply mobile-first responsive design patterns across all layouts and components.

- **Accessibility & SEO**:
  - `fixing-accessibility`: Audit and fix HTML accessibility issues including ARIA labels, keyboard navigation, focus management, color contrast, and form errors.
  - `fixing-metadata`: Audit and fix HTML metadata including page titles, meta descriptions, canonical URLs, Open Graph tags, and JSON-LD structured data.

---

**Execution Guidelines:**
- Before making frontend changes, identify which of these skills apply to the current task.
- Prioritize Next.js/React Native and Tailwind CSS patterns as specified in the project architecture.
- Rely on these specialized skill instructions rather than generic frontend knowledge to ensure the highest quality output for the Darna application.
